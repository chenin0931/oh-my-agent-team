OhMyAgentTeam CLI は、Web UI でできるほぼすべての操作をそのまま提供します（[イシュー](/issues)の作成、[エージェント](/agents)の割り当て、[デーモン](/daemon-runtimes)の起動など）。このページでは、すべてのトップレベルコマンドを 1 行の説明とともに一覧します。フラグや例の完全な一覧は `ohmyagentteam <command> --help` を実行してください。

## 認証する

CLI を初めて使うときにこのコマンドを実行して、**パーソナルアクセストークン（PAT）**を取得します。

```bash
omat login
```

ブラウザが自動的に開きます。Web アプリで承認すると、CLI が PAT（`mul_` プレフィックス付き）を `~/.ohmyagentteam/config.json` に保存します。これ以降のすべてのコマンドはこの PAT で認証されます。

<Callout type="tip">
CI やヘッドレス環境では、ブラウザフローをスキップできます。Web アプリの **Settings → Personal Access Tokens** で PAT を作成し、`omat login --token <mul_...>` で直接渡してください。
</Callout>

トークンの種類による違いについては、[認証とトークン](/auth-tokens)を参照してください。

## 認証とセットアップ

| コマンド | 用途 |
|---|---|
| `omat login` | ログインして PAT を保存 |
| `omat auth status` | 現在のログイン状態、ユーザー、ワークスペースを表示 |
| `omat auth logout` | ローカルの PAT を削除 |
| `omat setup cloud` | OhMyAgentTeam Cloud のワンショットセットアップ（ログイン + デーモンのインストール） |
| `omat setup self-host` | セルフホストバックエンドのワンショットセットアップ |

## ワークスペースとメンバー

| コマンド | 用途 |
|---|---|
| `omat workspace list` | アクセスできるすべてのワークスペースを一覧 |
| `omat workspace get <slug>` | 1 つのワークスペースの詳細を表示 |
| `omat workspace member list` | 現在のワークスペースのメンバーを一覧 |
| `omat workspace update <id> --name "..." [--description "..."] [--context "..."] [--issue-prefix "..."]` | ワークスペースのメタデータを更新（admin/owner）。長いフィールドは `--description-stdin` / `--context-stdin` を使用できます。 |

## イシューとプロジェクト

<Callout type="info">
`list` 系のコマンド（`omat issue list`、`autopilot list`、`project list` など）は、デフォルトで短く**そのままコピー＆ペーストできる** ID を出力します。イシューは `MUL-123` のようなイシューキー、それ以外のリソースは短い UUID プレフィックスです。以下の後続コマンドの `<id>` 引数は短い ID と完全な UUID のどちらも受け取るため、一般的な流れは `omat issue list` → キーをコピー → `omat issue get MUL-123` となります。正式な UUID が必要なときは `list` コマンドに `--full-id` を渡してください。
</Callout>

| コマンド | 用途 |
|---|---|
| `omat issue list` | イシューを一覧（コピー＆ペーストできるイシューキーを出力） |
| `omat issue get <id>` | 単一のイシューを表示（イシューキーまたは UUID を受け取る） |
| `omat issue create --title "..."` | 新しいイシューを作成 |
| `omat issue update <id> ...` | イシューを更新（ステータス、優先度、担当者など） |
| `omat issue assign <id> --to <name>` | メンバー、エージェント、またはスクワッドに割り当て（エージェントへの割り当ては実行をトリガー） |
| `omat issue status <id> <status>` | ステータス変更のショートカット |
| `omat issue search <query>` | キーワード検索 |
| `omat issue children <id>` | サブイシューを stage ごとに一覧 |
| `omat issue pull-requests <id>` | 紐付いた pull request と状態を一覧 |
| `omat issue runs <id>` | イシュー上のエージェント実行を表示 |
| `omat issue run-messages <task-id>` | 1 回の実行メッセージを表示 |
| `omat issue usage <id>` | イシュー単位の集計 token 使用量を表示 |
| `omat issue rerun <id>` | イシューの現在のエージェント担当者向けに新しいタスクを再キューイング |
| `omat issue cancel-task <task-id>` | キュー中または実行中のタスクをキャンセル |
| `omat issue comment <id> ...` | ネスト: コメントの表示 / 投稿 |
| `omat issue comment resolve/unresolve <comment-id>` | コメントスレッドを解決済み / 未解決にする |
| `omat issue subscriber <id> ...` | ネスト: 購読 / 購読解除 |
| `omat issue metadata <id> ...` | ネスト: イシューメタデータの読み書き |
| `omat issue label <id> ...` | ネスト: イシューのラベルを管理 |
| `omat project list/get/create/update/delete/status` | プロジェクトの CRUD |
| `omat label list/create/update/delete` | ワークスペースラベルの CRUD |

## エージェントとスキル

| コマンド | 用途 |
|---|---|
| `omat agent list` | ワークスペースのエージェントを一覧 |
| `omat agent get <slug>` | エージェントの構成を表示 |
| `omat agent create ...` | エージェントを作成 |
| `omat agent update <slug> ...` | エージェントを更新 |
| `omat agent archive <slug>` | アーカイブ |
| `omat agent restore <slug>` | アーカイブ済みのエージェントを復元 |
| `omat agent tasks <slug>` | エージェントのタスク履歴を表示 |
| `omat agent avatar <slug>` | エージェントのアバターをアップロード |
| `omat agent env <slug> ...` | エージェントの custom environment variables を読み取り / 置換 |
| `omat agent skills ...` | ネスト: スキルのアタッチ / デタッチ |
| `omat skill list/get/create/update/delete` | スキルの CRUD |
| `omat skill import ...` | GitHub、ClawHub、またはローカルマシンからスキルをインポート |
| `omat skill files ...` | ネスト: スキルのファイルを管理 |

### スキルインポートの競合

`omat skill import --url <url>` の既定値は `--on-conflict fail` です。同じ名前のスキルがすでに存在する場合、コマンドは構造化された `conflict` 結果で終了し、ワークスペースは変更されません。

既存スキルの作成者で、スキル ID とエージェントの紐付けを維持したまま内容を置き換える場合は `--on-conflict overwrite` を使います。既存スキルを残してコピーを取り込む場合は `--on-conflict rename` を使うと、`-2` のような接尾辞が自動で付きます。同名の項目を単に飛ばす場合は `--on-conflict skip` を使います。

```bash
omat skill import --url https://skills.sh/acme/repo/review-helper
omat skill import --url https://skills.sh/acme/repo/review-helper --on-conflict overwrite
omat skill import --url https://skills.sh/acme/repo/review-helper --on-conflict rename
omat skill import --url https://skills.sh/acme/repo/review-helper --on-conflict skip
```

## スクワッド

| コマンド | 用途 |
|---|---|
| `omat squad list` | ワークスペースのスクワッドを一覧 |
| `omat squad get <id>` | 単一のスクワッドを表示 |
| `omat squad create --name "..." --leader <agent>` | スクワッドを作成（owner / admin） |
| `omat squad update <id> ...` | 名前、説明、指示、リーダー、またはアバターを更新 |
| `omat squad delete <id>` | アーカイブ（ソフト削除） — 割り当て済みのイシューをリーダーに移管 |
| `omat squad member list/add/remove <squad-id>` | スクワッドメンバーを管理 |
| `omat squad activity <issue-id> <action\|no_action\|failed> --reason "..."` | スクワッドリーダーエージェントがターンごとに評価を記録するために使用 |

完全なモデルについては[スクワッド](/squads)を参照してください。

## オートパイロット

| コマンド | 用途 |
|---|---|
| `omat autopilot list` | ワークスペースのすべてのオートパイロットを一覧 |
| `omat autopilot get <id>` | 単一のオートパイロットを表示 |
| `omat autopilot create ...` | オートパイロットを作成 |
| `omat autopilot update <id> ...` | 更新 |
| `omat autopilot delete <id>` | 削除 |
| `omat autopilot runs <id>` | 実行履歴を表示 |
| `omat autopilot trigger <id>` | 手動で実行をトリガー |
| `omat autopilot trigger-add/update/delete <id>` | schedule または webhook trigger を管理 |
| `omat autopilot trigger-rotate-url <id> <trigger-id>` | webhook trigger URL をローテート |

`omat autopilot create/update` は、`create_issue` モードの autopilot が作成するイシューの既定購読者を設定する `--subscriber` も受け取ります。`update` では `--clear-subscribers` で削除できます。

## デーモンとランタイム

| コマンド | 用途 |
|---|---|
| `omat daemon start` | デーモンを起動（デフォルトはバックグラウンド。`--foreground` を追加するとフォアグラウンドで実行） |
| `omat daemon stop` | デーモンを停止 |
| `omat daemon restart` | デーモンを再起動 |
| `omat daemon status` | デーモンがオンラインかどうかと同時実行数を確認 |
| `omat daemon logs` | デーモンのログを表示 |
| `omat runtime list` | 現在のワークスペースのランタイムを一覧 |
| `omat runtime usage` | リソース使用量を表示 |
| `omat runtime activity` | 最近のアクティビティログ |
| `omat runtime update <id> ...` | ランタイム上で CLI 更新を開始 |
| `omat runtime delete <id> [--cascade]` | ランタイムを削除し、必要なら紐付くエージェントもアーカイブ |
| `omat runtime profile ...` | カスタム runtime profile とローカルパス上書きを管理 |

## 完全なフラグを確認する

すべてのコマンドが `--help` をサポートしています。

```bash
omat issue create --help
omat agent update --help
```

v2 では、各コマンドごとに専用の詳細なリファレンスページを提供する予定です。
